# 身份认证模块设计文档

## 概述

根据平台的认证权限配置需求，设计并实现了完整的身份认证管理模块，包括三个核心功能：认证配置、认证审核和认证日志。

## 认证等级体系

### 1. 未认证（None）
- **权限配置**：
  - 可入金：是
  - 可交易：是
  - 可提币：是
  - 提币限额：无限制
  - 需要证件：无

### 2. 普通认证（Basic）
- **权限配置**：
  - 可入金：是
  - 可交易：是
  - 可提币：是
  - 提币限额：10,000 USDT/日（可配置）
  - 需要证件：可配置（当前不需要上传证件）

### 3. 高级认证（Advanced）
- **权限配置**：
  - 可入金：是
  - 可交易：是
  - 可提币：是
  - 提币限额：无限制
  - 需要证件：身份证件（可配置是否需要收入证明、银行流水、手持身份证）

## 核心功能模块

### 1. 认证配置页面（VerificationConfigPage.vue）

**功能特点**：
- 三列卡片式布局，分别展示三个认证等级的配置
- 支持在线编辑各等级的权限配置：
  - 入金、交易、提币权限开关
  - 每日提币限额设置
  - 需要上传的证件类型选择
- 实时保存配置，立即对所有用户生效
- 配置说明文档，帮助管理员理解各等级差异

**路由**：`/users/verification-config`

### 2. 认证审核页面（VerificationAuditPage.vue）

**功能特点**：
- 统计概览：待审核、已通过、已拒绝、总申请数
- 搜索和筛选功能：
  - 按用户名、邮箱、姓名搜索
  - 按认证等级筛选
  - 按审核状态筛选
- 审核列表展示：
  - 用户基本信息
  - 当前等级 → 申请等级
  - 提交时间、审核时间
  - 审核状态
- 详情模态框：
  - 查看用户身份信息（姓名、身份证号、地址、职业等）
  - 预览上传的证件文档
  - 审核操作：通过、拒绝、要求补件
  - 填写审核意见

**路由**：`/users/verification-audit`

### 3. 认证日志页面（VerificationLogPage.vue）

**功能特点**：
- 统计概览：今日日志、总日志数、配置变更次数、审核操作次数
- 搜索和筛选功能：
  - 按用户名、操作人搜索
  - 按操作类型筛选
  - 按时间范围筛选（今天、最近一周、最近一月）
- 日志列表展示：
  - 操作时间、操作类型
  - 用户信息、等级变更
  - 操作人、操作描述
- 日志详情模态框：
  - 完整的操作信息
  - 等级变更前后对比
  - 详细的配置变更记录
- 导出功能：支持导出日志为CSV/Excel

**路由**：`/users/verification-log`

### 常量定义（constants/verification.js）

- **认证等级**：`VERIFICATION_LEVEL`（none, basic, advanced）
- **审核状态**：`VERIFICATION_STATUS`（pending, approved, rejected, resubmit）
- **证件类型**：`VERIFICATION_DOC_TYPE`（身份证、手持身份证、收入证明、银行流水、地址证明）
- **权限类型**：`PERMISSION_TYPE`（deposit, trade, withdraw）
- **日志操作类型**：`LOG_ACTION_TYPE`（配置更新、审核通过、审核拒绝、要求补件、等级升级、等级降级）

### Mock数据（mock/verification.js）

- **verificationConfig**：三个等级的权限配置数据
- **verificationAuditList**：认证审核记录列表（包含待审核、已通过、已拒绝、需补件等状态）
- **verificationLogList**：认证操作日志列表

## 使用流程

### 管理员配置认证规则
1. 进入"身份认证" → "认证配置"
2. 点击对应等级的"编辑配置"按钮
3. 调整权限开关、提币限额、证件要求
4. 保存配置

### 审核用户认证申请
1. 进入"身份认证" → "认证审核"
2. 查看待审核列表
3. 点击"查看详情"查看用户提交的信息和证件
4. 选择审核操作：
   - **通过**：用户认证等级升级
   - **拒绝**：申请被拒绝，填写拒绝原因
   - **要求补件**：提示用户补充材料

### 查看认证日志
1. 进入"身份认证" → "认证日志"
2. 使用筛选功能查找特定日志
3. 点击"详情"查看完整的操作记录
4. 可导出日志用于审计

## 扩展性

本模块设计具有良好的扩展性：

1. **添加新的认证等级**：在常量中定义新等级，在配置中添加对应规则
2. **添加更多证件类型**：在`VERIFICATION_DOC_TYPE`中定义新类型
3. **自定义权限项**：可以扩展权限配置，添加更多业务相关的权限控制
4. **集成API**：当前使用Mock数据，可轻松替换为真实API调用
5. **通知系统**：可以集成邮件/短信通知，在审核结果出来时通知用户

## 安全考虑

1. **敏感信息保护**：身份证号、地址等信息需要加密存储和传输
2. **审核权限控制**：只有特定角色的管理员才能进行审核操作
3. **操作日志完整**：所有认证相关操作都会记录日志，便于审计
4. **文件上传安全**：需要验证上传文件类型和大小，防止恶意文件
5. **防刷审核**：可以添加审核频率限制，防止滥用

## 未来优化方向

1. **智能审核**：集成OCR识别身份证信息，自动比对用户填写信息
2. **风险评分**：基于用户行为和历史数据，给出认证申请的风险评分
3. **批量审核**：支持批量通过或拒绝多个申请
4. **统计报表**：添加认证通过率、平均审核时长等统计图表
5. **移动端适配**：优化移动端审核体验
6. **自动化流程**：某些简单情况可以自动通过认证，减轻人工负担

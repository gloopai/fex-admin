# 汇率管理快速上手指南

## 📍 访问路径

**资金管理 → 汇率管理**

## 🎯 功能简介

汇率管理模块用于配置平台内所有交易对的汇率和手续费，包括：

- ✅ 支持多种数据源（币安、OKX、CoinGecko、自定义）
- ✅ 双向汇率自动映射，防止套利
- ✅ 灵活的分级费率体系
- ✅ 实时计算买入价和卖出价
- ✅ MFA 安全验证保障资金安全

## 🚀 快速开始

### 1️⃣ 新增第一个交易对

**步骤：**

1. 进入「汇率管理」页面
2. 点击右上角 **「+ 新增交易对」** 按钮
3. 填写基本信息：
   ```
   基础资产：USDT
   目标资产：BTC
   数据源：Binance
   汇率类型：浮动汇率
   市场汇率：0.000023
   启用该交易对：✓
   自动反向映射：✓
   ```

4. 展开 **「手续费与价差」** 配置：
   ```
   买入手续费率：0.5%
   卖出手续费率：0.5%
   ```
   系统会自动计算：
   - 平台买入价：0.000023115
   - 平台卖出价：0.000022885

5. 展开 **「分级费率设置」**（可选）：
   - 为不同等级用户设置优惠费率
   - VIP 用户可享受更低费率

6. 点击 **「保存」**，通过 MFA 验证后即可生效

---

### 2️⃣ 查看和筛选交易对

**筛选方式：**

- **状态筛选**：全部 / 已启用 / 已禁用

**列表信息展示：**

每个交易对卡片显示：

- 汇率类型（浮动/固定）
- 市场价、买入价、卖出价
- 买入/卖出手续费率
- 反向映射状态
- 最后更新时间
- 分级费率预览

## 💡 实战案例

### 案例 1：配置 USDT ↔ USD 汇率

**场景**：平台需要支持法币 USD 与稳定币 USDT 之间的兑换

**配置：**
```
基础资产：USD
目标资产：USDT
数据源：OKX
汇率类型：浮动汇率
市场汇率：1.0002
买入手续费率：0.3%
卖出手续费率：0.3%
```

**计算结果：**
- 平台买入价：1.0002 × (1 + 0.003) = 1.0032 USDT
- 平台卖出价：1.0002 × (1 - 0.003) = 0.9972 USDT

**场景**：用户可用 USDT 购买 ETH，或将 ETH 卖出换取 USDT

数据源：CoinGecko

卖出手续费率：0.8%
自动反向映射：开启

## 📊 费率计算公式

### 基本公式

```
平台买入价 = 市场汇率 × (1 + 买入手续费率)
平台卖出价 = 市场汇率 × (1 - 卖出手续费率)
```

### 示例计算

**已知：**
- 市场汇率：1 BTC = 43,500 USDT
- 买入费率：0.5%
- 卖出费率：0.5%

**计算：**
```
平台买入价 = 43,500 × (1 + 0.005) = 43,935 USDT
平台卖出价 = 43,500 × (1 - 0.005) = 43,065 USDT
点差 = 43,935 - 43,065 = 870 USDT
```

**平台收益：**
- 用户用 43,935 USDT 购买 1 BTC
- 用户卖出 1 BTC 获得 43,065 USDT
- 平台获利：870 USDT（约 2%）

## ⚠️ 注意事项

### 1. 防范套利
- ✅ 启用「自动反向映射」确保买卖价格合理
- ✅ 避免买入价 < 卖出价的情况
- ✅ 定期检查各交易对之间的关联关系

### 2. 及时更新
- ✅ 浮动汇率应设置合理的更新频率
- ✅ 关注外部数据源的稳定性
- ✅ 固定汇率需定期人工调整

### 3. 费率合理性
- ✅ 参考行业标准（通常 0.1%-1%）
- ✅ 考虑用户接受度和竞争力
- ✅ 平衡利润和用户体验

### 4. 安全操作
- ✅ 所有配置变更需通过 MFA 验证
- ✅ 建议双人复核重要配置
- ✅ 保留配置变更历史记录

### 5. 应急预案

## 🔧 常见问题

### Q1: 为什么我的买入价比卖出价低？
**A**: 这表示出现了价格倒挂，可能是：
- 买入费率设置过低或为负数
- 卖出费率设置过高
- 市场汇率异常

**解决方案**：检查费率设置，确保买入价 > 卖出价

---

### Q2: 分级费率如何生效？
**A**: 
- 系统根据用户的 VIP 等级自动应用对应费率
- 用户交易时，按当前等级的费率计算
- 用户等级提升后，立即享受新费率

---

### Q3: 自动反向映射的作用是什么？
**A**: 
- 当添加 A→B 的汇率时，自动创建 B→A 的反向汇率
- 防止两个方向汇率不一致导致的套利机会
- 保持汇率体系的完整性

---

### Q4: 如何选择合适的数据源？
**A**: 
- **Binance**: 流动性最好，价格最权威
- **OKX**: 亚洲时段数据质量好
- **CoinGecko**: 聚合多个交易所，价格平均
- **Custom**: 无可靠数据源时使用

建议：主流交易对用 Binance，长尾交易对用 CoinGecko

---

### Q5: 固定汇率和浮动汇率有什么区别？
**A**: 
- **浮动汇率**: 跟随外部数据源实时更新
- **固定汇率**: 保持设定值不变，需手动调整

适用场景：
- 浮动：主流币种、高流动性交易对
- 固定：长尾币种、缺乏数据源的交易对

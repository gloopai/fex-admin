# 管理台客服操作面板设计

## 目标

在现有管理台增加一个可直接处理前台客服会话的 Demo 工作台。前台用户发送消息后，管理台能够查看会话、人工回复、记录内部备注并结束会话；管理台回复可在同一浏览器来源的前台页面实时显示。

本阶段以完整演示闭环为目标，不接入后端、WebSocket、第三方客服系统，也不实现多人客服协作。

## 原型范围调整

实现阶段按用户反馈收敛为轻量演示原型。保留三栏展示、前后台本地消息联动、文字回复、结束会话和简单内部备注；不实现复杂状态筛选、接单流程、管理台图片回复或多客服操作。后文较完整的产品化描述作为未来扩展参考，不属于本次原型验收条件。

## 方案选择

采用三栏客服工作台：左侧会话列表，中间聊天处理区，右侧用户资料与内部备注。相比双栏加抽屉，该布局能同时呈现待处理队列、对话上下文和用户信息；相比表格加详情页，更适合连续处理实时消息。

## 导航与路由

- 管理台新增一级菜单“客服管理”。
- 工作台路由为 `/admin/customer-service/workbench`。
- 菜单使用独立客服图标，并遵循现有管理台导航和路由模式。
- 本阶段沿用管理台当前的访问机制，不额外设计客服角色、细粒度权限或排班权限。

## 页面结构

### 会话列表

左栏承担会话发现与筛选：

- 顶部展示全部、待回复、处理中和已结束数量。
- 支持关键词搜索、状态筛选和仅看未读。
- 会话项展示用户头像、邮箱或 UID、最后一条消息、更新时间、未读数和状态。
- 新用户消息将会话提升到列表顶部。
- 点击会话即打开处理，不设置额外“接单”步骤；打开时清除管理台未读数。
- 没有匹配会话时展示带重置筛选操作的空状态。

### 聊天处理区

中栏承担人工回复：

- 顶部展示用户标识、会话状态、开始时间和“结束会话”操作。
- 消息区区分用户与客服消息，显示文字、图片和时间。
- 底部提供常用回复、文字输入、图片附件和发送按钮。
- 客服首次回复时，会话自动进入“处理中”。
- 客服回复后，会话进入“待用户回复”；用户再次发送消息后恢复为“待回复”。
- 会话已结束或未选择会话时禁用输入。
- 结束会话需要一次确认，结束后历史消息仍可查看。

### 用户信息区

右栏展示处理当前问题所需的演示信息：

- UID、邮箱、VIP、KYC、信用分、注册时间和账户状态。
- 历史会话次数、当前会话开始时间和最近登录时间。
- 仅管理台可见的内部备注，支持编辑和保存。
- 不提供资产调整、身份修改或其他与客服目标无关的高风险操作。

窄屏管理台保留中间聊天区为主视图，会话列表通过左侧面板切换，用户信息通过右侧抽屉打开，避免三栏被压缩到不可用宽度。

## 会话模型

会话包含：

- `id`：会话唯一标识。
- `userId`：前台用户 UID。
- `user`：邮箱、昵称、VIP、KYC、信用分、注册时间、最近登录时间和账户状态的演示快照。
- `status`：`waiting`、`active`、`waiting-user` 或 `closed`。
- `messages`：按创建时间排列的消息列表。
- `adminUnread`：管理台未读数。
- `userUnread`：前台未读数。
- `internalNote`：仅管理台可见备注。
- `createdAt`、`updatedAt`、`closedAt`：会话时间字段。

消息包含 `id`、`sender`、`text`、`imageDataUrl` 和 `createdAt`。`sender` 只允许 `user` 或 `agent`。

## 状态规则

- 前台发送第一条消息时创建 `waiting` 会话。
- 前台在未结束会话中发送消息时，状态变为 `waiting`，管理台未读加一。
- 管理台打开会话只清除管理台未读，不改变会话状态。
- 管理台发送回复时状态变为 `waiting-user`，前台未读加一。
- 管理台可将会话显式标记为 `active`，用于表示正在处理但尚未回复。
- 管理台结束会话后状态变为 `closed`，输入禁用并记录 `closedAt`。
- 用户在已结束会话后再次发送消息时创建新会话，不复用已结束会话。

## 演示数据与同步

共享客服仓储封装会话读写、迁移、订阅和业务操作，前台与管理台不直接操作存储结构。

- 使用同源 `localStorage` 持久化 Demo 会话。
- 同标签页通过自定义事件通知更新。
- 多标签页通过 `storage` 事件同步。
- 每次写入后重新读取并发出快照，页面只消费不可变快照。
- 首次进入管理台工作台且没有任何数据时，可生成少量示例历史会话；真实前台会话使用当前登录账号信息。
- 初始化时补齐缺失字段并忽略结构损坏的数据，确保旧 Demo 数据不会使页面崩溃。

未来接入真实服务时，保留仓储接口，将实现替换为 HTTP 历史查询、消息发送接口和 WebSocket 订阅。

## 前台调整

- 移除当前固定延迟的自动客服回复。
- 页面初始化时恢复当前用户最近的未结束会话，没有时仅显示欢迎消息。
- 用户发送文字或图片后写入共享仓储。
- 管理台回复通过仓储订阅实时追加到前台并清除前台未读。
- 管理台结束会话后，前台展示“本次服务已结束”；用户继续发送时自动创建新会话。
- 前台仍要求登录访问。

## 图片处理

- 仅接受图片 MIME 类型。
- 单张图片限制为 1 MB，超过限制时在输入区显示错误。
- 图片转为 Data URL 后写入 Demo 存储，保证刷新与跨标签页后仍可显示。
- 捕获存储配额错误并提示用户清理演示会话或改发文字。

## 异常与边界

- 空白文字且没有图片时不能发送。
- 会话结束后不能从管理台继续发送。
- 当前会话因另一标签页结束时，管理台立即禁用输入并更新状态。
- 搜索和筛选不会修改会话数据。
- 没有选中会话、数据损坏、图片读取失败和存储失败均提供明确空态或错误提示。
- 同时写入采用最后一次写入优先的 Demo 语义；不声称具备生产级冲突解决能力。

## 测试与验收

自动测试覆盖：

- 创建、恢复和结束会话。
- 用户消息与客服消息的状态流转。
- 管理台与前台未读计数。
- 结束后用户新建会话。
- 内部备注仅存在于会话管理数据中。
- 搜索、状态筛选和统计。
- 损坏数据修复与空消息拒绝。
- 管理台路由与菜单配置。

浏览器验收覆盖：

1. 前台用户发送消息。
2. 管理台会话列表出现新会话和未读数。
3. 管理台直接打开会话、发送人工回复并保存内部备注。
4. 前台实时显示回复。
5. 管理台结束会话，前台显示结束状态。
6. 前台再次发送后生成新会话。
7. 检查桌面三栏布局与窄屏面板切换。

## 非目标

- 后端接口、WebSocket 服务和数据库。
- 多客服抢单、分配、转接、排班和在线状态。
- SLA、机器人、工单、评价、敏感词和消息撤回。
- 生产级并发控制、附件对象存储和长期历史归档。
- 客服细粒度权限与审计日志。

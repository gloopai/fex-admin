# 费率模板功能说明

## 📋 概述

费率模板功能允许管理员创建可复用的手续费和分级费率配置模板，快速应用到不同的汇率兑换项目中，大大提高配置效率并确保费率政策的一致性。

## 🎯 核心优势

### 1. **快速配置**
- 一键应用预设的费率模板
- 避免重复手动输入
- 减少配置错误

### 2. **标准化管理**
- 统一的费率政策
- 便于全局调整和优化
- 支持分类管理（标准/优惠/VIP/自定义）

### 3. **灵活复用**
- 一个模板可应用于多个交易对
- 支持模板使用统计
- 方便追踪模板应用情况

### 二、创建/编辑模板

#### 三个配置区域

**1. 基本信息**
- 模板名称（如：标准费率模板）
- 模板类型（标准/优惠/VIP/自定义）
- 描述说明（适用场景）
- 启用状态

**2. 基础手续费与价差**
- 买入手续费率 (%)
- 卖出手续费率 (%)
- 修改后自动同步到所有用户等级

**3. 分级费率设置**
- BASIC（普通用户）
- SILVER（白银会员）
- GOLD（黄金会员）
- PLATINUM（白金会员）
- VIP（至尊会员）
- 每个等级可独立设置买入/卖出费率
- 支持重置为默认值

#### 操作流程

```
1. 点击「+ 新增模板」
   ↓
2. 展开「基本信息」
   - 填写模板名称
   - 选择模板类型
   - 编写描述说明
   ↓
3. 展开「基础手续费与价差」
   - 设置基础买入费率
   - 设置基础卖出费率
   - 自动同步到各级别
   ↓
4. 展开「分级费率设置」（可选）
   - 微调各个等级的费率
   - 实现差异化定价
   ↓
5. 点击「保存」→ MFA 验证 → 完成
```

   - 可以手动修改基础费率

   - 修改后不再受模板影响

4. **保存配置**
   - 通过 MFA 验证
   - 保存到交易对配置

### 关键代码逻辑

#### 1. 应用模板函数

```javascript
const applyFeeTemplate = (templateId) => {
  const template = feeTemplates.value.find(t => t.id === templateId)
  if (template) {
    form.feeTemplateId = templateId
    form.buyMarkup = template.baseMarkup.buy
    form.sellMarkup = template.baseMarkup.sell
    form.userLevelRates = JSON.parse(JSON.stringify(template.userLevelRates))
    calculateRates()
  }
}
```

**功能**:
- 根据模板 ID 查找模板
- 自动填充基础费率
- 自动填充分级费率
- 重新计算买入价和卖出价

---

#### 2. 初始化分级费率

```javascript
const initUserLevelRates = () => {
  form.userLevelRates = {
    [USER_LEVEL_TIER.BASIC]: { 
      buy: form.baseMarkup.buy, 
      sell: form.baseMarkup.sell 
    },
    [USER_LEVEL_TIER.SILVER]: { 
      buy: form.baseMarkup.buy * 0.8, 
      sell: form.baseMarkup.sell * 0.8 
    },
    [USER_LEVEL_TIER.GOLD]: { 
      buy: form.baseMarkup.buy * 0.6, 
      sell: form.baseMarkup.sell * 0.6 
    },
    [USER_LEVEL_TIER.PLATINUM]: { 
      buy: form.baseMarkup.buy * 0.4, 
      sell: form.baseMarkup.sell * 0.4 
    },
    [USER_LEVEL_TIER.VIP]: { 
      buy: form.baseMarkup.buy * 0.2, 
      sell: form.baseMarkup.sell * 0.2 
    }
  }
}
```

**功能**:
- 基于基础费率自动生成各级别费率
- 使用预设的折扣系数（80%、60%、40%、20%）
- 保持费率结构的合理性

---

#### 3. 更新基础费率同步

```javascript
const updateBaseMarkup = () => {
  initUserLevelRates()
}
```

**功能**:
- 当基础费率变化时，自动重新计算各级别费率
- 保持费率层级关系的一致性

## 📈 使用统计

### 模板使用情况

系统会记录每个模板的使用情况：

- **usageCount**: 已应用于多少个交易对
- 在模板列表中清晰展示
- 便于评估模板的实用价值

### 热门模板分析

根据使用次数，可以识别出：

1. **最常用模板**: usageCount 最高
2. **闲置模板**: usageCount = 0，可以考虑优化或禁用
3. **潜力模板**: 新创建但增长迅速

### 1. 模板命名规范

**推荐格式**: `{场景} + {费率水平} + 模板`

**示例**:
- ✅ 标准费率模板
- ✅ 优惠费率模板
- ✅ VIP 专属模板
- ✅ 高收益定制模板

---

### 2. 模板分类策略

**按用户群体分类**:
- 标准模板 → 普通用户
- 优惠模板 → 活跃用户
- VIP 模板 → 高端用户

**按交易对分类**:
- 主流交易对 → 低费率
- 长尾交易对 → 高费率
- 新兴交易对 → 促销费率

---

### 3. 费率梯度设计

**推荐梯度**（以标准模板为例）:

```
BASIC:      100% 基准费率
SILVER:      80% 基准费率（优惠 20%）
GOLD:        60% 基准费率（优惠 40%）
PLATINUM:    40% 基准费率（优惠 60%）
VIP:         20% 基准费率（优惠 80%）
```

**优势**:
- 梯度明显，激励升级
- 结构清晰，便于理解
- 利润可控，风险分散

---

### 4. 模板更新策略

**何时更新模板**:

✅ **应该更新**:
- 平台整体费率政策调整
- 市场竞争需要降价
- 运营成本变化

❌ **不应频繁更新**:
- 小幅随机调整
- 临时性促销
- 个别交易对特殊需求

**更新注意事项**:
1. 提前通知受影响的用户
2. 评估对现有交易对的影响
3. 考虑创建新版本而非直接修改
4. 保留历史版本用于回滚

---

### 5. 模板组合策略

**多模板配合使用**:

```
场景 1: 新交易对上线
→ 选择「标准费率模板」
→ 观察市场反应
→ 如有需要调整为「优惠费率模板」

场景 2: 促销活动
→ 临时应用「优惠费率模板」
→ 活动结束后恢复「标准费率模板」

场景 3: 用户分层运营
→ 普通用户交易对 → 「标准费率模板」
→ VIP 用户交易对 → 「VIP 专属模板」
```

## 📝 常见问题

### Q1: 修改模板会影响已应用的交易对吗？

**A**: 
- ❌ **不会直接影响**。交易对在应用模板时会复制当时的费率配置
- ✅ 如需批量更新，需要手动或使用批量工具

**设计理念**:
- 保持交易对配置的独立性
- 避免模板变更影响既有业务
- 给予管理员完全的控制权

---

### Q2: 如何批量更新多个交易对的费率？

**方案 A**: 手动逐个更新
- 进入每个交易对的编辑页面
- 重新选择模板
- 保存

**方案 B**: 开发批量工具（推荐）
- 选择要更新的交易对列表
- 选择目标模板
- 批量应用

**方案 C**: API 批量调用
- 通过后端 API 批量更新
- 适合技术管理员

---

### Q3: 可以删除已使用的模板吗？

**A**: 
- ⚠️ **不建议删除**正在使用的模板
- ✅ 可以先禁用模板，然后逐步迁移交易对
- ✅ 或者创建新模板替代旧模板

**最佳实践**:
1. 禁用到期模板而非删除
2. 保留历史记录便于追溯
3. 归档长期不用的模板

---

### Q4: 如何查看某个模板被哪些交易对使用？

**当前版本**:
- 仅显示使用数量（usageCount）
- 暂不支持详细列表查询

**后续优化**:
- 增加「查看使用详情」功能
- 列出所有使用该模板的交易对
- 提供快速跳转链接

---

### Q5: 模板能否导出和导入？

**当前版本**:
- ❌ 不支持导出/导入功能

**后续优化**:
- ✅ 支持 JSON 格式导出
- ✅ 支持批量导入模板
- ✅ 支持模板分享功能

# 入金手动确认与公共地址 MFA 设计

## 范围

- 入金审核详情保留现有“选择链上通知”流程，并增加“手动输入确认”模式。
- 手动模式仅填写 TxHash、付款地址、实际到账金额，三项均必填。
- 币种、网络、平台地址继承入金申请，不允许手工修改。
- 公共收款地址的新增、编辑、启用和停用均在实际写入前通过现有 MFA 验证弹窗。

## 入金手动确认

审核区使用两个模式按钮。链上通知模式继续要求选择未确认通知记录；手动模式显示三个输入框。提交时校验文本非空、金额大于零、TxHash 未被已有链上记录或其他已入账订单使用。

Mock API 接收可选的 `manualConfirmation`。有效时生成来源为 `manual` 的关联记录，使用订单自身的币种、网络、平台地址及最低确认数，并将订单状态更新为已入账。详情中显示“人工确认”。

## 公共收款地址 MFA

页面先保存待执行操作，不直接修改 Mock 仓库。MFA 验证成功后执行新增、编辑或状态切换；取消 MFA 时丢弃待执行操作。原有换址保留历史和整链唯一规则保持不变。

## 错误处理与测试

- 手动字段缺失、金额无效或 TxHash 重复时阻止入账并显示错误提示。
- MFA 未验证或取消时不得修改公共地址列表。
- 自动化测试覆盖手动入账数据、重复哈希拒绝、页面三字段和公共地址 MFA 接入。

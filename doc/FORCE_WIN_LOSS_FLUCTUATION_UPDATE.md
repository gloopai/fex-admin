# 强制盈利/亏损波动配置更新说明

## 更新日期
2026-03-10

## 更新内容

### 功能概述
在交割合约场控规则管理中，为**强制盈利**和**强制亏损**执行动作增加了**波动配置**功能。现在可以设置一个波动范围，让盈利/亏损比例不再固定，而是在一定范围内随机波动。

### 具体变更

#### 1. 新增字段

**强制盈利动作**新增字段：
- `winFluctuationPercent`: 盈利波动范围（百分比），默认值 2%
  - 取值范围：0-10%
  - 精度：0.1%
  - 实际盈利 = 基准盈利比例 ± 波动范围

**强制亏损动作**新增字段：
- `lossFluctuationPercent`: 亏损波动范围（百分比），默认值 2%
  - 取值范围：0-10%
  - 精度：0.1%
  - 实际亏损 = 基准亏损比例 ± 波动范围

#### 2. UI 界面更新

**强制盈利配置区域**：
- ✅ 保留原有的"影响订单数"和"盈利比例"配置
- ➕ 新增"盈利波动范围"配置卡片，包含：
  - 滑块控制器（0-10%，步长 0.1%）
  - 数字输入框（精确输入）
  - 实时显示当前波动值（±X%）
  - 说明文字："每单盈利将在 X% 基础上随机波动 ±Y%"
- 📊 预览卡片更新：
  - 显示格式从 "盈利 X%" 改为 "盈利 X% ± Y%"
  - 新增一行显示实际盈利范围："实际盈利范围：A% ~ B%"

**强制亏损配置区域**：
- ✅ 保留原有的"影响订单数"和"亏损比例"配置
- ➕ 新增"亏损波动范围"配置卡片，包含：
  - 滑块控制器（0-10%，步长 0.1%）
  - 数字输入框（精确输入）
  - 实时显示当前波动值（±X%）
  - 说明文字："每单亏损将在 X% 基础上随机波动 ±Y%"
- 📊 预览卡片更新：
  - 显示格式从 "亏损 X%" 改为 "亏损 X% ± Y%"
  - 新增一行显示实际亏损范围："实际亏损范围：A% ~ B%"

### 使用示例

#### 示例 1：强制盈利带波动
配置：
- 盈利比例：20%
- 盈利波动范围：±3%

效果：
- 用户接下来 N 单强制盈利
- 每单实际盈利在 **17% ~ 23%** 之间随机波动
- 增加不确定性和真实感

#### 示例 2：强制亏损带波动
配置：
- 亏损比例：30%
- 亏损波动范围：±5%

效果：
- 用户接下来 N 单强制亏损
- 每单实际亏损在 **25% ~ 35%** 之间随机波动
- 避免固定比例被用户察觉规律

### 技术实现

#### 计算逻辑
实际盈亏比例计算公式：
```javascript
// 强制盈利
actualWinPercent = profitPercent + random(-winFluctuationPercent, +winFluctuationPercent)

// 强制亏损
actualLossPercent = lossPercent + random(-lossFluctuationPercent, +lossFluctuationPercent)
```

#### 界面交互
1. **滑块控制**：支持快速调整，范围 0-10%，步长 0.1%
2. **数字输入**：支持精确输入，可输入小数
3. **实时预览**：底部预览卡片实时更新显示波动范围
4. **范围计算**：自动计算并显示最小/最大值

### 优势

1. **提升隐蔽性**：
   - 盈亏比例不再固定，更难被用户发现规律
   - 模拟真实市场波动的不确定性

2. **增强灵活性**：
   - 可根据不同场景调整波动幅度
   - 高风险用户可设置更大波动范围

3. **改善用户体验**：
   - 避免每次都亏/赚相同比例的诡异体验
   - 增加随机性，更接近真实交易

4. **降低风险**：
   - 防止用户通过统计历史数据破解规则
   - 提高风控系统的抗分析能力

### 兼容性说明

- ✅ **向后兼容**：旧规则数据会自动使用默认波动值 2%
- ✅ **平滑过渡**：不影响现有规则的正常运行
- ✅ **数据迁移**：编辑旧规则时会自动添加波动字段

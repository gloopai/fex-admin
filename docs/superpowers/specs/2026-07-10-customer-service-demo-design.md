# 客服演示功能设计

## 目标

在现有前台增加一个必须登录后使用的客服演示功能。页面参考用户提供的移动端截图，同时适配项目当前的桌面端深色视觉体系。此阶段只实现浏览器内交互，不连接客服后端，不持久化聊天记录，也不上传附件。

## 范围

本次包含：

- 新增独立客服页面 `/front/customer-service`。
- 在顶部导航、个人中心菜单和前台全局右下角悬浮按钮增加客服入口。
- 支持发送文字、选择图片并在消息区本地预览。
- 用户发送消息后，通过短暂延迟展示预设客服回复。
- 为路由和所有入口应用现有前台登录鉴权及登录后回跳逻辑。
- 提供桌面端和移动端响应式布局。

本次不包含：

- HTTP、WebSocket 或第三方客服系统对接。
- 服务端消息、已读状态、会话分配和历史记录。
- 附件上传、压缩或云端存储。
- 客服后台工作台和管理功能。

## 方案

采用“独立客服页 + 三处统一入口”。相比全站抽屉或桌面抽屉与移动页面混合的方案，该方案只有一套聊天状态和布局，能复用现有 Vue Router 鉴权与登录回跳机制，且便于后续把演示消息层替换为真实客服接口。

## 页面与组件

### 客服页面

新增独立 Vue 页面，职责包括：

- 展示返回按钮和“客服”标题。
- 初始化客服欢迎消息“你好，请问有什么需要帮助的吗？”。
- 展示客服消息、用户文字消息和用户图片消息。
- 管理输入框、图片选择、发送状态及自动滚动。
- 在演示回复等待期间防止重复生成回复。

移动端按照参考图使用占满视口的聊天布局和固定底部输入区。桌面端沿用前台深色背景，在居中的有限宽度聊天面板内承载相同内容。返回按钮优先返回浏览器上一页；没有可用历史时回到前台首页。

### 输入区

输入区包含文本输入、附件按钮和发送按钮：

- 只有文字非空或已选择图片时才能发送。
- Enter 发送，Shift+Enter 换行。
- 图片通过隐藏的文件输入选择，并用对象 URL 在本地预览。
- 发送后清空当前输入和待发送图片。
- 页面卸载时释放尚未释放的对象 URL。

### 演示回复

用户发送任意有效消息后，页面等待一个短暂固定时长，再追加一条客服预设回复。回复内容说明客服已收到消息并正在处理。回复完全由页面内状态产生，刷新页面后恢复到初始欢迎消息。

## 入口与鉴权

- 顶部导航新增“客服”入口。
- 个人中心桌面侧栏与移动快捷菜单新增“在线客服”。
- 前台全局布局增加右下角客服悬浮按钮；客服页面自身隐藏该按钮，避免重复入口。
- 客服路由声明 `requiresAuth: true`。
- 未登录用户点击任一入口或直接访问 URL 时，复用现有登录页重定向参数；登录成功后回到客服页。

## 状态与数据流

聊天状态仅存在于客服页面组件内：

1. 页面创建时加入欢迎消息。
2. 用户输入文字或选择图片。
3. 发送操作把当前内容转换为用户消息并清空编辑区。
4. 页面滚动到最新消息。
5. 定时器追加客服演示回复，再次滚动到底部。

入口只负责导航，不共享或持久化聊天状态。后续接入真实服务时，可将消息发送与接收抽离为 composable 或 store，而无需改变入口和路由结构。

## 异常与边界

- 空白文字且没有图片时不发送。
- 文件选择器限制为图片类型；无法读取预览时不创建消息。
- 自动回复定时器在页面卸载时清理，避免离开页面后更新状态。
- 长文本自动换行，图片限制最大显示尺寸，聊天区域独立滚动。
- 底部输入区考虑移动端安全区域，避免被系统手势区域遮挡。

## 验证

- 运行生产构建，确认新增页面和路由可编译。
- 验证未登录直接访问客服页会跳转登录，并携带正确回跳地址。
- 验证登录后顶部导航、个人中心菜单、移动快捷菜单和悬浮按钮均能进入客服页。
- 验证空消息不可发送，文字消息可发送，Enter 与 Shift+Enter 行为正确。
- 验证图片选择、本地预览、发送后清空以及客服延迟回复。
- 分别检查桌面与移动视口下的聊天滚动、固定输入区、返回按钮和悬浮入口显隐。
